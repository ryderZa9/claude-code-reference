★Quick Start — Top 10 for New Agentic Engineers

/plan copy

Enter plan mode — Claude proposes changes for your review before touching anything. Use this for every non-trivial task.

/init copy

Bootstraps CLAUDE.md in your project so Claude understands your codebase rules, stack, and preferences from day one.

/memory copy

Edit your persistent memory files. What Claude remembers across sessions lives here.

/compact copy

Compress the conversation when context gets long. Keeps Claude sharp without starting over.

/diff copy

Interactive viewer for all uncommitted changes. Browse per-turn diffs with arrow keys.

/permissions copy

Control exactly what Claude can do automatically vs. what requires your approval.

/mcp copy

Connect external tools — Slack, Jira, Google Drive, GitHub — into Claude's context.

/batch copy

Spawn parallel agents to execute large-scale changes across your codebase simultaneously.

/cost copy

See token usage and spending at any point. Stay on top of your budget.

/doctor copy

Diagnose installation issues instantly. Run this first when something feels off.

⊛Permission Modes

Cycle through modes with Shift+Tab. The mode you pick determines how much Claude does autonomously vs. how much requires your approval.

ModeCycle KeyBehaviorRisk
default Shift+Tab ×1 Claude asks permission before each action. Safest for new projects. low
acceptEdits Shift+Tab ×2 Claude auto-accepts file edits without prompting. Still asks for shell commands. medium
plan Shift+Tab ×3 Claude drafts all changes in plan mode for your review before any execution. low
auto Shift+Tab ×4 ML classifier auto-approves safe operations. Still requires approval for dangerous actions.
Requires: Team/Enterprise/API plan + Sonnet 4.6+ or Opus 4.6
 medium
bypassPermissions Shift+Tab ×5 Skip all permission prompts. Use only in sandboxed/trusted environments. high

⚑CLI Flags

Flags for the claude startup command. Combine freely.

FlagDescriptionExample
-p, --print Print response without interactive mode (SDK/scripting mode) claude -p "what does this do?"
-c, --continue Continue most recent conversation in current directory claude -c
-r, --resume Resume specific session by ID or name claude -r "my-task"
-n, --name Set display name for this session claude -n "auth-refactor"
-w, --worktree Start in an isolated git worktree branch claude -w feature-x
--model Set model at startup (aliases: sonnet, opus) claude --model opus
--effort Set effort level at startup claude --effort high
--permission-mode Begin in a specific permission mode claude --permission-mode plan
--remote Create a web session on claude.ai claude --remote "Fix auth bug"
--ide Auto-connect to IDE if available claude --ide
--add-dir Grant file access to additional directories at startup claude --add-dir ../shared-lib
--system-prompt Replace the entire system prompt claude --system-prompt "You are..."
--append-system-prompt Append to the default system prompt claude --append-system-prompt "Always use TypeScript"
--chrome Enable Chrome browser integration claude --chrome
--verbose Enable verbose logging claude --verbose
--debug Enable debug mode with optional filters claude --debug "api,mcp"
--bare Skip auto-discovery for faster startup (use with -p) claude --bare -p "query"
-v, --version Show current version claude -v
--help Show help claude --help
